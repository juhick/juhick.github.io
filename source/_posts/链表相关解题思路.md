# 链表

## 递归反转链表的一部分（递归）

### 递归反转整个链表

```cpp
ListNode reverse(ListNode head) {
    if (head.next == null) return head;
    ListNode last = reverse(head.next);
    head.next.next = head;
    head.next = null;
    return last;
}
```

<!-- more -->

具体来说，我们的 `reverse` 函数定义是这样的：

**输入一个节点** **`head`****，将「以** **`head`** **为起点」的链表反转，并返回反转之后的头结点**。

明白了函数的定义，在来看这个问题。比如说我们想反转这个链表：

![img](b9fff89cb1f69f93555c158bee3adce9ca8d7487.jpg)

那么输入`reverse(head)`后，会在这里进行递归：

```cpp
ListNode last = reverse(head.next);
```

不要跳进递归（你的脑袋能压几个栈呀？），而是要根据刚才的函数定义，来弄清楚这段代码会产生什么结果：

![img](c244744301acfdaae7b1a26bc92d0d63748ad675.jpg)

这个 `reverse(head.next)` 执行完成后，整个链表就成了这样：

![img](be3eed15a20254b8d77d0f33798da8f9bf081b7b.jpg)

并且根据函数定义，`reverse` 函数会返回反转之后的头结点，我们用变量 `last` 接收了。

现在再来看下面的代码：

```cpp
head.next.next = head;
```

接下来：

return last;

### 反转链表的前N个结点

![img](df31967e270f60cb760c9c14bcae2e16dd4f7e14.jpg)

解决思路与反转整个链表差不多，只要稍加修改即可：

```cpp
ListNode successor = null; // 后驱节点

// 反转以 head 为起点的 n 个节点，返回新的头结点
ListNode reverseN(ListNode head, int n) {
    if (n == 1) { 
        // 记录第 n + 1 个节点
        successor = head.next;
        return head;
    }
    // 以 head.next 为起点，需要反转前 n - 1 个节点
    ListNode last = reverseN(head.next, n - 1);

    head.next.next = head;
    // 让反转之后的 head 节点和后面的节点连起来
    head.next = successor;
    return last;
}
```

![img](f962eeb29ecfd905bf9935d167455fd0f96e365f.jpg)

### 反转链表的一部分

```cpp
ListNode reverseBetween(ListNode head, int m, int n) {
    // base case
    if (m == 1) {
        return reverseN(head, n);
    }
    // 前进到反转的起点触发 base case
    head.next = reverseBetween(head.next, m - 1, n - 1);
    return head;
}
```

### 总结

递归的思想相对迭代思想，稍微有点难以理解，处理的技巧是：不要跳进递归，而是利用明确的定义来实现算法逻辑。

处理看起来比较困难的问题，可以尝试化整为零，把一些简单的解法进行修改，解决困难的问题。

值得一提的是，递归操作链表并不高效。和迭代解法相比，虽然时间复杂度都是 O(N)，但是迭代解法的空间复杂度是 O(1)，而递归解法需要堆栈，空间复杂度是 O(N)。所以递归操作链表可以作为对递归算法的练习或者拿去和小伙伴装逼，但是考虑效率的话还是使用迭代算法更好。

## 如何K个一组反转链表（迭代）

### 反转一个链表的元素

```cpp
// 反转以 a 为头结点的链表
ListNode reverse(ListNode a) {
    ListNode pre, cur, nxt;
    pre = null; cur = a; nxt = a;
    while (cur != null) {
        nxt = cur.next;
        // 逐个结点反转
        cur.next = pre;
        // 更新指针位置
        pre = cur;
        cur = nxt;
    }
    // 返回反转后的头结点
    return pre;
}
```

![img](19872a9373104e6de4ff98d7c6fb3e9c64b0e885.gif)

### 反转一个区间的元素

```cpp
/** 反转区间 [a, b) 的元素，注意是左闭右开 */
ListNode reverse(ListNode a, ListNode b) {
    ListNode pre, cur, nxt;
    pre = null; cur = a; nxt = a;
    // while 终止的条件改一下就行了
    while (cur != b) {
        nxt = cur.next;
        cur.next = pre;
        pre = cur;
        cur = nxt;
    }
    // 返回反转后的头结点
    return pre;
}
```

### K个一组翻转链表

```cpp
ListNode reverseKGroup(ListNode head, int k) {
    if (head == null) return null;
    // 区间 [a, b) 包含 k 个待反转元素
    ListNode a, b;
    a = b = head;
    for (int i = 0; i < k; i++) {
        // 不足 k 个，不需要反转，base case
        if (b == null) return head;
        b = b.next;
    }
    // 反转前 k 个元素
    ListNode newHead = reverse(a, b);
    // 递归反转后续链表并连接起来
    a.next = reverseKGroup(b, k);
    return newHead;
}
```

## 如何判断回文链表

### 寻找回文串

从中间向两端扩展

```cpp
string palindrome(string& s, int l, int r) {
    // 防止索引越界
    while (l >= 0 && r < s.size()
            && s[l] == s[r]) {
        // 向两边展开
        l--; r++;
    }
    // 返回以 s[l] 和 s[r] 为中心的最长回文串
    return s.substr(l + 1, r - l - 1);
}
```

因为回文串长度可能为奇数也可能是偶数，长度为奇数时只存在一个中心点，而长度为偶数时存在两个中心点，所以上面这个函数需要传入`l`和`r`。

### 判断回文串

不需要考虑奇偶情况

从两端向中间逼近

```cpp
bool isPalindrome(string s) {
    int left = 0, right = s.length - 1;
    while (left < right) {
        if (s[left] != s[right])
            return false;
        left++; right--;
    }
    return true;
}
```

### 判断回文单链表

输入一个单链表的头结点，判断这个链表中的数字是不是回文

可以通过模拟双指针来实现回文判断的功能：

```cpp
// 左侧指针
ListNode left;

boolean isPalindrome(ListNode head) {
    left = head;
    return traverse(head);
}

boolean traverse(ListNode right) {
    if (right == null) return true;
    boolean res = traverse(right.next);
    // 后序遍历代码
    res = res && (right.val == left.val);
    left = left.next;
    return res;
}
```

这么做的核心逻辑是什么呢？**实际上就是把链表节点放入一个栈，然后再拿出来，这时候元素顺序就是反的**，只不过我们利用的是递归函数的堆栈而已。

![img](245e396c68dcaf769d72e909bcb98a4b97c4cefe.gif)

#### 优化空间复杂度

1. 先通过「双指针技巧」中的快慢指针来找到链表的中点：

   ```cpp
   ListNode slow, fast;
   slow = fast = head;
   while (fast != null && fast.next != null) {
       slow = slow.next;
       fast = fast.next.next;
   }
   // slow 指针现在指向链表中点
   ```

   ![img](e3f9643f57342c88edd155a8af2b11ab280977cc.jpg)

2. 如果`fast`指针没有指向`null`，说明链表长度为奇数，`slow`还要再前进一步：

   ```cpp
   if (fast != null)
       slow = slow.next;
   ```

   ![img](828ec0ae419fa1d7634bb10e4868abc74ac6d492.jpg)

3. 从`slow`开始反转后面的链表，现在就可以开始比较回文串了：

   ```cpp
   ListNode left = head;
   ListNode right = reverse(slow);
   
   while (right != null) {
       if (left.val != right.val)
           return false;
       left = left.next;
       right = right.next;
   }
   return true;
   ```

   ![img](b5162e61baa5d8c38b4292c73d1acec03fce6a1f.jpg)

至此，把上面 3 段代码合在一起就高效地解决这个问题了，其中`reverse`函数很容易实现：

```cpp
ListNode reverse(ListNode head) {
    ListNode pre = null, cur = head;
    while (cur != null) {
        ListNode next = cur.next;
        cur.next = pre;
        pre = cur;
        cur = next;
    }
    return pre;
}
```

![img](19872a9373104e6de4ff98d7c6fb3e9c64b0e885.gif)

只要在函数 return 之前加一段代码即可恢复原先链表顺序：

```cpp
p.next = reverse(q);
```
